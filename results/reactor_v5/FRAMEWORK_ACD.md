## 0. QUÉ ES ESTO

OMNI-MIND ya no es un "motor de preguntas". Lo que emerge de las últimas sesiones es un **framework completo de medicina cognitiva para sistemas** — capaz de diagnosticar, explicar, prescribir, ejecutar y verificar transformaciones cognitivas en cualquier dominio.

El nombre propuesto es **Álgebra Cognitiva Diagnóstica (ACD)** porque:
- **Álgebra** — tiene operaciones, composiciones, leyes, invariantes, y las operaciones no son conmutativas ni asociativas (confirmado en sesiones previas)
- **Cognitiva** — opera sobre el aparato de pensamiento del sistema, no sobre sus síntomas
- **Diagnóstica** — su función primaria es diagnosticar Y prescribir, no solo describir

## 2. EL MODELO CAUSAL COMPLETO

### 2.1 La cadena causal de 4 niveles

```
NIVEL 1: REPERTORIO COGNITIVO
         ¿Qué herramientas tiene el sistema?
         → Inventario de INT×P×R activas, atrofiadas, ausentes, fosilizadas
                    │
                    │ Leyes IC1-IC7
                    ▼
NIVEL 2: DISTRIBUCIÓN DE LENTES
         ¿Qué produce ese repertorio?
         → Cada INT×P×R genera S, Se, o C preferentemente
         → La distribución del repertorio = distribución de lentes
                    │
                    │ Leyes D1-D5, umbrales empíricos
                    ▼
NIVEL 3: ESTADO DIAGNÓSTICO
         ¿En qué estado está el sistema?
         → 4 estados de equilibrio (E1 muerte, E2 latencia, E3 funcional, E4 pleno)
         → 6 perfiles de desequilibrio (operador, visionario, zombi, genio, autómata, potencial)
         → Flags de peligro (autómata oculto, intoxicación F2→F3, monopolio Se)
                    │
                    │ Leyes C1-C5, T1-T6, recetas
                    ▼
NIVEL 4: PERFIL FUNCIONAL (SÍNTOMA OBSERVABLE)
         ¿Qué hace el sistema?
         → 7 funciones × 3 lentes = 21 scores observables
         → Relaciones entre funciones activas/inactivas
         → Comportamiento bajo tensión (modo B+N)
```

### 2.2 La inversión diagnóstica

El framework opera en DOS DIRECCIONES:

```
DIRECCIÓN DESCENDENTE (explicación):
  INT×P×R → ¿por qué este perfil de lentes? → ¿por qué estas funciones?
  "El sistema es operador ciego PORQUE solo tiene herramientas de S"

DIRECCIÓN ASCENDENTE (diagnóstico desde observación):
  Funciones observadas → perfil de lentes → repertorio cognitivo inferido
  "Observo S↑ Se↓ C↓ → operador ciego → probablemente INT{S} + P{S} + R{S}"
```

La dirección descendente es EXPLICATIVA (entiende causas).  
La dirección ascendente es DIAGNÓSTICA (infiere desde síntomas).  
Ambas son necesarias. El Motor usa la ascendente para diagnosticar y la descendente para prescribir.

## 5. EL PIPELINE COMPLETO: DE OBSERVACIÓN A INTERVENCIÓN

```
══════════════════════════════════════════════════════════════════
FASE A: DIAGNÓSTICO (ascendente — de síntoma a causa)
══════════════════════════════════════════════════════════════════

PASO 0: OBSERVACIÓN
  Input:  Caso (negocio, paciente, sistema, equipo, producto...)
  Output: Descripción del estado actual

PASO 1: PERFIL FUNCIONAL (Nivel 4 — síntoma)
  - Evaluar 7 funciones × 3 lentes = 21 scores
  - Identificar relaciones activas entre funciones
  - Verificar F2→F3(B): si F2>0.60 y F3<0.25 → flag intoxicación
  
PASO 2: PERFIL DE LENTES (Nivel 3 — efecto)
  - Calcular S_avg, Se_avg, C_avg
  - Calcular gradiente = promedio(S, Se, C)
  - Calcular gap = max(S,Se,C) - min(S,Se,C)
  
PASO 3: ESTADO DIAGNÓSTICO (Nivel 3)
  - Si gap < 0.15 → estado de equilibrio (E1-E4 según gradiente)
  - Si gap ≥ 0.15 → perfil de desequilibrio (6 perfiles según distribución)
  - Verificar flags:
    · S>0.60 + C>0.60 + Se<0.30 → AUTÓMATA ETERNO (peligro oculto)
    · Una F con Se>0.70 + resto Se<0.30 → MONOPOLIO Se (rigidez)
    · Se_avg < 0.25 → ZONA TÓXICA

PASO 4: DIAGNÓSTICO COGNITIVO (Nivel 1 — causa)  ★NUEVO★
  - Inferir qué INT están activas/atrofiadas
  - Inferir qué P usa el sistema por defecto
  - Inferir qué R emplea habitualmente
  - Verificar: ¿hay monopolio INT? (IC2)
  - Verificar: ¿hay desacoples INT-P? (IC3) ¿INT-R? (IC4)
  - Verificar: ¿faltan pares complementarios? (IC5)
  - Verificar: ¿hay R aislados sin validación cruzada? (IC6)

══════════════════════════════════════════════════════════════════
FASE B: PRESCRIPCIÓN (descendente — de causa a intervención)
══════════════════════════════════════════════════════════════════

PASO 5: SELECCIÓN COGNITIVA ★NUEVO★
  - Según perfil → seleccionar INT×P×R que generan la lente faltante
  - Consultar tabla maestra de transformación (gradientes_equilibrio_inteligencias.md §4)
  - Verificar compatibilidad INT-P (IC3) e INT-R (IC4)
  - Asegurar pares complementarios (IC5) y validación cruzada (IC6)
  
PASO 6: SELECCIÓN DE TIPO LÓGICO Y MODO
  - Falta S → Nivel 1 (Conductas) → MOVER/GENERAR
  - Falta Se → Nivel 3-5 (Criterios/Gobierno) → ENMARCAR/PERCIBIR
  - Falta C → Nivel 4 (Reglas) → GENERAR/ANALIZAR

PASO 7: SECUENCIACIÓN
  - SIEMPRE: Se primero → S después → C último (Ley C4)
  - NUNCA empezar por F7 o F2 sin Se previo
  - Si gap > 0.30 → NO recomendar escalar (F7)
  - Si Se_avg < 0.40 → NO recomendar replicar (F7)

══════════════════════════════════════════════════════════════════
FASE C: EJECUCIÓN (las preguntas como programa compilado)
══════════════════════════════════════════════════════════════════

PASO 8: COMPILAR PROGRAMA DE PREGUNTAS
  - Las INT seleccionadas determinan QUÉ preguntas
  - Los P seleccionados determinan CÓMO se organizan
  - Los R seleccionados determinan CÓMO se responden
  - El Gestor de la Matriz compila el programa
  - El Motor vN ejecuta el programa contra el caso

PASO 9: EJECUTAR ENJAMBRE
  - Tier según complejidad (Principio 31: Reflejo→Respuesta→Análisis→Profundo→Cartografía)
  - Múltiples modelos cubren diferentes territorio de la Matriz
  - Estigmergia entre agentes del enjambre

══════════════════════════════════════════════════════════════════
FASE D: VERIFICACIÓN (cierre del ciclo)
══════════════════════════════════════════════════════════════════

PASO 10: EVALUAR RESULTADO
  - ¿Se movió la lente faltante? (delta ≥ +0.25 para Se)
  - ¿Se amplió el repertorio cognitivo? (más INT×P×R activas)
  - ¿Disminuyó el gap entre lentes? (<0.30)
  - ¿Subió el gradiente? (hacia E3 o E4)
  - ¿Se activaron ≥3 funciones con Se>0.40? (umbral de cierre)
  - ¿Emergió algún desacople nuevo?

PASO 11: DECIDIR
  - Si cierre → documentar. Pasar a vigilancia (E4 homeostasis).
  - Si inerte → loop con inteligencias de refuerzo. Verificar si la secuencia fue correcta.
  - Si tóxico → PARAR. Verificar si se amplificó lente dominante (D1).
```

## 6. LA MATRIZ COMO ESTRUCTURA ALGEBRAICA

### 6.1 Formulación algebraica actualizada

La Matriz original era: **3 Lentes × 7 Funciones × 18 Inteligencias**

Con los hallazgos de esta sesión, la Matriz completa es:

```
M = L³ × F⁷ × INT¹⁸ × P¹⁵ × R¹²
```

Donde:
- L³ = espacio de lentes (S, Se, C) ∈ [0,1]³
- F⁷ = espacio de funciones (F1...F7), cada una evaluada en L³
- INT¹⁸ = espacio de inteligencias activas/inactivas
- P¹⁵ = espacio de pensamientos activos/inactivos
- R¹² = espacio de razonamientos activos/inactivos

### 6.2 Las operaciones del álgebra

| Operación | Qué hace | Ejemplo |
|-----------|----------|---------|
| EXTRAER | Observar con las INT seleccionadas | INT-17 extrae: "¿valores declarados = vividos?" |
| CRUZAR | Combinar observaciones de múltiples INT | INT-17 × INT-03: "¿el gap existencial = gap estructural?" |
| LENTES | Evaluar cada observación desde S, Se, C | "¿ese gap opera (S)? ¿se comprende (Se)? ¿persiste (C)?" |
| INTEGRAR | Componer funciones con lentes | F3.Se + F5.Se → cadena de sentido |
| ABSTRAER | Subir de nivel lógico | De conducta (Nivel 1) a criterio (Nivel 3) |
| FRONTERA | Delimitar lo que pertenece al sistema | F5: "¿esto es nuestro problema o del entorno?" |

### 6.3 Propiedades algebraicas confirmadas

- **No conmutatividad:** INT-17 antes de INT-16 ≠ INT-16 antes de INT-17 (Se primero ≠ S primero)
- **No asociatividad:** (INT-17 ∘ INT-03) ∘ INT-16 ≠ INT-17 ∘ (INT-03 ∘ INT-16)
- **Dirección canónica:** Se → S → C (Ley C4, invariante)
- **Idempotencia de saturación:** n=2 pases óptimos para la mayoría de INT
- **Elemento absorbente:** Si Se_avg < 0.25, cualquier operación que no inyecte Se es inerte o tóxica

### 7.2 Diferencial competitivo

Ningún framework existente combina:
1. Diagnóstico a nivel de herramientas cognitivas (no solo síntomas)
2. Prescripción específica (no "mejora tu comunicación" sino "activa INT-17 + P05 + R03")
3. Predicción cuantitativa con umbrales empíricos
4. Aplicabilidad cross-dominio validada (negocio, salud, STEM, educación, deporte, derecho, personal)
5. Detección de peligro oculto (el autómata eterno es invisible para métricas convencionales)

El framework tiene una inversión más profunda que INT×P×R → Lentes:

```

    │
    │ patrones recurrentes de preguntas cristalizan en...
    ▼

    │
    │ cada combinación genera preferentemente...
    ▼
LENTES (S, Se, C)
    │
    │ la distribución de lentes produce...
    ▼
FUNCIONES × LENTES (síntoma observable)
```

**Las 18 inteligencias no son un catálogo cerrado. Son patrones que EMERGIERON de tipos de preguntas.** INT-17 (Existencial) existe porque hay un tipo de pregunta recurrente — "¿por qué existe esto?" — que aparece cross-dominio y que genera Se de una forma específica. Si mañana un dominio produce un tipo de pregunta que no encaja en ninguna INT existente, eso ES una nueva inteligencia cristalizándose.

Lo mismo para P y R: P05 (Primeros principios) es un patrón recurrente de organización cognitiva que emergió de preguntas tipo "¿qué es irreduciblemente verdad aquí?". R03 (Abducción) es un patrón inferencial que emergió de preguntas tipo "¿cuál es la mejor explicación para esto?".

### 10.4 El mecanismo de cristalización

¿Cuándo un "tipo de pregunta" se convierte en una nueva INT, P, o R?

```
CRITERIO DE CRISTALIZACIÓN:

1. RECURRENCIA: El tipo de pregunta aparece en ≥3 dominios distintos
2. IRREDUCTIBILIDAD: No puede descomponerse en combinación de INT/P/R existentes
3. LENTE ESPECÍFICA: Genera una lente (S, Se, o C) de una forma diferenciable
4. FUNCIÓN ESPECÍFICA: Opera sobre una función (F1-F7) de forma diferenciable
5. PREGUNTAS PROPIAS: Tiene un repertorio de ≥5 preguntas que SOLO esta herramienta genera
```

Ejemplo: ¿Podría existir INT-19 (Temporal)?
- ¿Genera preguntas cross-dominio? Sí: "¿cuándo es el momento correcto?", "¿qué cambia con el tiempo?", "¿hay ventana de oportunidad?"
- ¿Es irreductible? Parcialmente: INT-13 (Prospectiva) cubre futuro, pero no cubre timing ni ritmo ni ciclos.
- ¿Genera lente específica? Sí: C (continuidad) con matiz temporal que INT-13 no tiene.
- Resultado: Candidata a cristalización. Necesita validación empírica.

El Reactor es el GENERADOR de tipos de preguntas → el cristalizador de nuevas herramientas cognitivas.

### 10.7 Diseño ad hoc de herramientas cognitivas

Cualquier dominio puede requerir herramientas cognitivas a medida. El framework ACD permite:

1. **INT ad hoc:** Si un dominio genera preguntas de observación únicas, se diseña una inteligencia específica. Ejemplo para fisioterapia: "INT-Propioceptiva" → preguntas sobre consciencia corporal del paciente que van más allá de INT-10 (Cinestésica).

2. **P ad hoc:** Si un dominio requiere una estrategia organizativa específica. Ejemplo para negocio de servicios: "P-Experiencial" → pensamiento que organiza desde la experiencia del cliente, no desde la operación interna.

3. **R ad hoc:** Si un dominio requiere un mecanismo inferencial no cubierto. Ejemplo para salud: "R-Etiológico" → razonamiento que infiere cadena causal de síntoma a causa pasando por múltiples sistemas interconectados.

**El procedimiento de diseño:**

```
1. IDENTIFICAR: ¿Qué preguntas del dominio no encajan en el catálogo actual?
2. AGRUPAR: ¿Forman un patrón coherente? (≥5 preguntas relacionadas)
3. CLASIFICAR: ¿Es INT (observación), P (organización), o R (inferencia)?
4. MAPEAR LENTE: ¿Qué lente genera preferentemente? (S, Se, C)
5. MAPEAR FUNCIÓN: ¿Sobre qué función opera? (F1-F7)
6. TESTEAR IRREDUCTIBILIDAD: ¿Se puede expresar como combinación de existentes?
7. Si irreductible → CRISTALIZAR como nueva herramienta L1
8. Si reducible → ANOTAR como variante de dominio de herramienta existente
```
